What is DASK?

DASK is a Python framework to provide simple access to parallel processing and distributed computing.

More details, please!

You are writing Python scripts which take longer than your lifetime to run on your laptop and you are looking for the easiest way to speed up the execution by running the process on multiple CPU cores? You want your application to run on a cluster of computers but you don't want to be bothered by the implementation details of parallel computation? Setting up a computing cluster is nothing you want to deal with in order to run your Python program with decent speed? Seems like DASK is the tool you are looking for!

But I don't have two dozen computers at home to build my own cluster.

Don't worry, you are not alone with this problem. The computer pool from the physics faculty at KIT is a collection of 34 PCs which are all accessible via an SSH interface. Furthermore on every computer the user home directories are synchronised through a network drive: The files are the same across all devices. With some configuration, which is provides on this website, and reasonable modifications to a Python program the computers can be used to enhance computationally heavy tasks on large datasets. The DASK framework takes the responsibility to distribute the workload among the nodes in this cluster and manages all the resulting effort.

Which rules do I have to follow when I want to use the computers?

When you registered for an account you already agreed to the computing rules which can be found here to look up. This implies that using a DASK cluster on some / all the computers is allowed if it is used for a purpose that has something to do with your studies. Illegal actions like mining for cryptographic currencies or downloading copyrighted media from the internet are going to be detected and the user will be taken into responsibility. In case you have any concerns you can always ask the administrator or the "poolraum-hiwi" for their advice.

Furthermore we ask you to care about other users. The computers are also used for tutorials and practical curses which cannot take place when they are overloaded. Therefore we want you to limit the resource allocation on every node as it is done in the examples provided on this website so that the computers are not completely taken by a single person. If you still need more power you can do your computation with higher resource allocation overnight when no user is sitting in front of the devices but pay attention that everything is done until the morning. When this is still not enough for you: Sorry but you should look for another platform to run your code, maybe go back to the idea of building your own cluster?

DASK Setup

Step 1: Install Software

First the DASK python libraries need to be installed. Installing all of them at once is the easiest and most intrusive way to add the framework to your Python environment.

pip install dask[complete]

You can also install only the components you need but then you have to make sure that all your dependencies are available, of which the first one needs to be installed in any ways:

  • pip install dask[distributed]: Install DASK with the tools for distributed computing, this is necessary!
  • pip install dask[array]: Install DASK and numpy
  • pip install dask[dataframe]: Install DASK, numpy and pandas
  • pip install dask[bag]: Install DASK and cloudpickle
  • pip install dask: Install only DASK. This might be useful if you only want to install the task scheduler.

There are also other optional dependencies, described in the docs. The dependency distributed is necessary and must be installed. You can select multiple optional dependencies by separating them with a comma ,,e.g.: pip install dask[array,distributed]

When you install these packages on one of the PCs from the computing pool then they are going to be available on all of the machines. Make sure you install them in you user directory. If pip starts to complain then add the --user option which tells it to use your home directory as the target location for the installation.

Unfortunately the package paramiko sometimes does not get installed automatically. Therefore it is always useful to manually install it in addition to make sure it is available:

pip install paramiko

Step 2: Cluster Configuration

Let's talk about the not recommended but in this case necessary way to set up a cluster: Using only SSH. For more information about establishing an SSH connection please take a look into the Instructions for using SSH for the physics computer pool. After you know how to establish an SSH connection you have to setup the passwordless login to remote devices via the SSH key. Doing this by calling the enable_ssh_key_pool.sh script is highly recommended if you don't know what you are doing.
